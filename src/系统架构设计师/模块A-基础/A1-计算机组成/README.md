## 1. 计算机系统基本组成

### 1.1 冯·诺依曼体系结构

```
┌─────────────────────────────────┐
│           运算器（ALU）           │
│  算术运算：+、-、×、÷            │
│  逻辑运算：与、或、非、异或        │
└──────────┬──────────┬────────────┘
           │          │
┌──────────┴──┐  ┌────┴──────────┐
│  控制器（CU） │  │   存储器      │
│  指令译码    │  │   主存+辅存    │
│  时序控制    │  │               │
└──────┬──────┘  └───────────────┘
       │
┌──────┴──────┐
│  输入设备    │
│  输出设备    │
└─────────────┘
```

**五大部件**：运算器、控制器、存储器、输入设备、输出设备

- 运算器 + 控制器 = **CPU（中央处理器）**
- 主存储器（内存）+ 辅助存储器（外存）= **存储器**
- 输入设备 + 输出设备 = **I/O 设备**

### 1.2 CPU 结构

**运算器（ALU）** 包含：
- 算术逻辑单元（ALU）：执行算术和逻辑运算
- 累加寄存器（AC）：存放运算结果或源操作数
- 状态寄存器（PSW）：存放运算状态（进位、溢出、零标志等）
- 数据缓冲寄存器（DR）：暂存从主存取来的数据
- 地址寄存器（MAR）：存放要访问的主存单元地址

**控制器（CU）** 包含：
- 程序计数器（PC）：存放下一条要执行的指令地址
- 指令寄存器（IR）：存放正在执行的指令
- 指令译码器（ID）：对指令进行译码分析
- 时序发生器：产生时序信号

### 1.3 指令系统

**指令格式**：
```
┌────────────┬────────────┐
│ 操作码（OP） │ 操作数地址   │
│  指令做什么   │  对谁做      │
└────────────┴────────────┘
```

**寻址方式**（考试重点）：

| 寻址方式 | 地址计算 | 特点 |
|----------|----------|------|
| 立即寻址 | 操作数直接在指令中 | 速度最快，灵活性最差 |
| 直接寻址 | EA = A | 简单，一次访存 |
| 间接寻址 | EA = (A) | 两次访存，速度慢 |
| 寄存器寻址 | EA = Ri | 速度快，寄存器数量有限 |
| 寄存器间接寻址 | EA = (Ri) | 灵活 |
| 基址寻址 | EA = BR + A | 利于程序重定位 |
| 变址寻址 | EA = IX + A | 利于数组/循环处理 |
| 相对寻址 | EA = PC + A | 利于浮动代码 |

> ⭐ **考试常考**：区分基址寻址 vs 变址寻址
> - 基址寻址：基址寄存器内容由操作系统管理，面向**多道程序**
> - 变址寻址：变址寄存器内容由用户设定，面向**用户**

### 1.4 CISC vs RISC

| 对比项 | CISC（复杂指令集） | RISC（精简指令集） |
|--------|-------------------|-------------------|
| 指令数量 | 多（>200） | 少（<100） |
| 指令长度 | 可变长 | 固定长 |
| 寻址方式 | 多种 | 少（Load/Store） |
| 执行时间 | 差异大，大部分多周期 | 大部分单周期 |
| 流水线 | 难以实现 | 易于实现 |
| 代表 | x86（Intel/AMD） | ARM、MIPS、RISC-V |

## 2. 存储系统

### 2.1 存储层次结构（速度↑ 容量↓ 成本↑）

```
    寄存器（最快，容量最小）
       ↓
    高速缓存（Cache L1/L2/L3）
       ↓
    主存储器（RAM）
       ↓
    辅助存储器（SSD/HDD）
       ↓
    海量存储（磁带/光盘）
```

### 2.2 Cache（高速缓存）

**工作原理**：
- CPU 先访问 Cache，命中则直接读取（快）
- 未命中则访问主存，同时将该块调入 Cache（慢）

**性能指标**：
- **命中率 H** = Cache 命中次数 / 总访问次数
- **平均访问时间** = H × t_cache + (1-H) × t_memory
- 其中 t_cache 为 Cache 访问时间，t_memory 为主存访问时间

**映射方式**（⭐重点）：

| 映射方式 | 特点 | 优缺点 |
|----------|------|--------|
| 直接映射 | 主存块只能映射到 Cache 固定行 | 计算简单，冲突多 |
| 全相联映射 | 主存块可映射到 Cache 任意行 | 冲突少，硬件复杂，成本高 |
| 组相联映射 | 分组直接映射，组内全相联 | 折中方案，最常用 |

**计算公式**：
```
直接映射：Cache 行号 = 主存块号 mod Cache 行数
全相联：需要逐行比较标签
组相联：组号 = 主存块号 mod 组数，组内全相联查找
```

**替换算法**（Cache 和虚拟内存都考）：

| 算法 | 策略 | 说明 |
|------|------|------|
| FIFO | 先进先出 | 可能产生 Belady 异常 |
| LRU | 最近最少使用 | 性能好，需要硬件支持 |
| LFU | 最不经常使用 | 按访问频率 |
| OPT | 最优替换 | 理论最优，不可实现，用于对比 |

### 2.3 虚拟存储器

**基本概念**：
- 将程序逻辑地址空间与物理内存分离
- 程序看到的地址空间远大于实际物理内存
- 不常用的页面放在磁盘上，需要时再调入

**地址变换过程**：
```
逻辑地址 → 页号 + 页内偏移
                ↓
           查页表 → 物理页框号 + 页内偏移 = 物理地址
                ↓
           若缺页 → 从磁盘调入
```

**页面置换算法**（⭐重点，几乎每年考）：

1. **OPT（最优）**：淘汰以后最长时间不使用的页面（理论最优）
2. **FIFO**：淘汰最先进入内存的页面
   - ⚠️ **Belady 异常**：增加物理块数反而增加缺页次数
3. **LRU（最近最少使用）**：淘汰最近最久未访问的页面
4. **LFU（最不经常使用）**：淘汰访问次数最少的页面
5. **CLOCK（时钟算法/NRU）**：循环扫描，访问位为 0 则淘汰

> ⭐ **常考题型**：给定页面访问序列，计算不同算法的缺页率

**例题**：
```
页面访问序列：7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
物理块数：3

FIFO 缺页次数：15 次
LRU  缺页次数：12 次
OPT  缺页次数：9 次
```

### 2.4 局部性原理

- **时间局部性**：刚被访问的数据很可能很快再次被访问（循环）
- **空间局部性**：被访问数据附近的数据很可能也被访问（数组顺序访问）

这是 Cache 和虚拟存储器设计的理论基础。

## 3. 流水线技术

### 3.1 流水线基本概念

将指令执行过程分为多个阶段，每个阶段由不同部件并行处理。

```
指令1: [取指] → [译码] → [执行] → [访存] → [写回]
指令2:          [取指] → [译码] → [执行] → [访存] → [写回]
指令3:                   [取指] → [译码] → [执行] → [访存] → [写回]
```

### 3.2 流水线性能计算（⭐重点）

**基本公式**：
```
流水线周期 = 最长阶段的时间（瓶颈阶段）
执行时间 = 第一条指令完成时间 + (n-1) × 流水线周期
         = k × t + (n-1) × t    （k 为阶段数，t 为流水线周期）
         = (k + n - 1) × t

吞吐率 TP = n / 总执行时间
加速比 S  = 不用流水线的时间 / 用流水线的时间 = k × n × t / (k+n-1) × t
效率 E    = k × t / (k+n-1) × t = k / (k+n-1)
```

**例题**：
```
流水线分为 5 个阶段，各阶段时间分别为 80ns、70ns、90ns、80ns、70ns
流水线周期 = max(80, 70, 90, 80, 70) = 90ns
完成 10 条指令需要 = (5 + 10 - 1) × 90 = 1260ns
吞吐率 = 10 / 1260 ≈ 7.94 × 10⁹ 条/秒
```

### 3.3 流水线冲突

| 冲突类型 | 原因 | 解决方法 |
|----------|------|----------|
| 资源冲突（结构冲突） | 多条指令争用同一资源 | 增加资源（如哈佛结构分离指令/数据存储） |
| 数据冲突 | 后续指令依赖前面指令的结果 | 数据旁路（转发）、插入气泡、指令重排 |
| 控制冲突 | 转移指令改变执行流 | 分支预测、延迟转移 |

## 4. 总线系统

### 4.1 总线分类

- **内部总线**：CPU 内部，连接寄存器、ALU
- **系统总线**：连接 CPU、内存、I/O 接口
  - 数据总线（DB）：传输数据，宽度决定一次传输位数
  - 地址总线（AB）：传输地址，宽度决定寻址空间
  - 控制总线（CB）：传输控制信号
- **外部总线**：连接主机与外部设备（USB、PCIe 等）

### 4.2 总线仲裁

- **集中仲裁**：总线控制器集中管理（链式查询、计数器定时查询、独立请求）
- **分布式仲裁**：各设备竞争总线使用权

## 5. 系统可靠性计算

### 5.1 串联与并联系统

**串联系统**（所有部件都正常系统才正常）：
```
R = R1 × R2 × ... × Rn
```

**并联系统**（所有部件都故障系统才故障）：
```
R = 1 - (1-R1) × (1-R2) × ... × (1-Rn)
```

**混合系统**：先算局部，再逐步合并

### 5.2 可靠性指标

- **MTBF**（平均无故障时间）= 1 / 故障率 λ
- **MTTR**（平均修复时间）
- **可用性** = MTBF / (MTBF + MTTR)

> 例：系统可用性 99.99% 意味着全年停机时间 = 365 × 24 × 60 × (1 - 0.9999) ≈ 52.6 分钟

### 5.3 N 模冗余

- **N 版本程序设计**：N 个不同团队独立开发同一功能，投票决定结果
- **三模冗余（TMR）**：3 个模块，多数表决（2/3 表决）

## 6. 校验码（数据编码）

### 6.1 奇偶校验
- 奇校验：数据位 + 校验位中 1 的个数为奇数
- 偶校验：数据位 + 校验位中 1 的个数为偶数
- 只能检奇数位错误，不能纠错

### 6.2 CRC 循环冗余校验（⭐重点）
- 基于多项式除法（模 2 除法）
- 能检错不能纠错
- **计算方法**：
  1. 确定生成多项式 G(x)，得到 r 位（r = G(x) 最高次幂）
  2. 在数据后面加 r 个 0
  3. 用模 2 除法除以 G(x)，余数即为 CRC 校验码
  4. 将余数附加到数据后发送

### 6.3 海明码（⭐重点）
- 能检错也能纠错（1 位纠错）
- **校验位数量**：2^k ≥ n + k + 1（n 为数据位，k 为校验位）
- **校验位位置**：放在 2 的幂次位置（1, 2, 4, 8, 16...）
- 每个校验位负责校验特定的数据位
